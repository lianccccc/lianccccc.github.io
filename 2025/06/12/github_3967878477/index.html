Metasploit Framework DNS Spoofing Vulnerability

漏洞信息

漏洞名称： Metasploit Framework DNS Spoofing Vulnerability

漏洞类型： 服务器端请求伪造

漏洞等级： 中危

漏洞描述： Metasploit Framework中的DNS欺骗模块（bailiwicked_domain和bailiwicked_host）存在异常抛出问题，可能导致DNS欺骗攻击失败。此问题与DNS库的更改未正确移植到这些模块中有关。

产品厂商： Rapid7

产品名称： Metasploit Framework

来源： https://github.com/rapid7/metasploit-framework/issues/20172

类型： rapid7/metasploit-framework:github issues

来源概述

Both bailiwicked modules are throwing exceptions on 7b5b57a392927e8ddac9824ff1713d051de2a3ff. I’m guessing that the issue is related to DNS library changes that weren’t ported into these modules.

Steps to reproduce

I was able to trigger these exceptions without a proper-vulnerable target, just by setting the options sufficiently to run the module.

These two modules should also probably be updated to default their SRCPORT to 0 since that’s a supported value to select an port automatically.

Spotted while testing #20145.
