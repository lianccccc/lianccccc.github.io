Fortinet SSL-VPN 会话管理漏洞

漏洞信息

漏洞名称： Fortinet SSL-VPN 会话管理漏洞

漏洞编号：

  • CVE： CVE-2024-50562

漏洞类型： 权限绕过

漏洞等级： 中危

漏洞描述： CVE-2024-50562是Fortinet SSL-VPN门户中的一个会话管理漏洞，认证用户的会话在注销操作期间未正确失效。这使得会话cookie在显式注销后仍然有效并可被重用，可能导致会话劫持攻击。漏洞的核心问题在于SSL-VPN注销过程中的会话清理不足，特别是主会话cookie（SVPNCOOKIE）和临时会话cookie（SVPNTMPCOOKIE）在注销时可能未在服务器端正确失效。

CVE-2024-50562 - Technical Analysis for PSIRT

Vulnerability Overview

CVE-2024-50562 is a session management vulnerability in Fortinet SSL-VPN portals where authenticated user sessions are not properly invalidated during logout operations. This allows session cookies to remain valid and be reused after explicit logout, enabling potential session hijacking attacks.

CVSS Score: 4.4 (Medium)
Advisory: FG-IR-24-339

Vulnerability Mechanism

The core issue lies in insufficient session cleanup during the SSL-VPN logout process. Specifically:

  1. Session Cookie Persistence: Primary session cookies (SVPNCOOKIE) and temporary session cookies (SVPNTMPCOOKIE) may not be properly invalidated server-side during logout
  2. Authentication Bypass: Valid session cookies can be reused in new browser sessions to access authenticated portal areas without re-authentication
  3. Session State Confusion: The server fails to recognize that a logout event should invalidate all associated session tokens

Testing Methodology

This scanner reproduces the vulnerability through a 4-phase authentication lifecycle test:

Phase 1: Initial Authentication

Key Indicators of Success:

  • Server returns ret=1 in response body
  • Presence of /remote/hostcheck_install in response
  • Session cookies (SVPNCOOKIE, SVPNTMPCOOKIE) are set

Phase 2: Session Cookie Analysis

Phase 3: Explicit Logout

Expected Secure Behavior:

  • All session cookies should be explicitly cleared
  • Server should invalidate session state server-side
  • Subsequent requests with old cookies should fail

Phase 4: Session Reuse Validation

Vulnerability Detection Logic

The scanner determines vulnerability status through response content analysis:

Vulnerable System (CVE-2024-50562 Present):

  • Response: Authenticated portal content is served
  • Indicators: Direct access to portal dashboard/content
  • Cookie State: Original session cookies remain valid
  • Verdict: REUSED - Session hijacking possible

Secure System (Patched/Not Vulnerable):

  • Response: Login form or authentication challenge
  • Indicators: Presence of name="username" input fields or /remote/login redirects
  • Cookie State: Session cookies invalidated server-side
  • Verdict: INVALIDATED - Proper session management

Security Implications

Attack Scenarios:

  1. Session Hijacking: Attacker captures session cookies post-logout
  2. Credential Harvesting: Malicious scripts preserve authentication state
  3. Shared Terminal Risk: Sessions remain active on shared/public computers

Business Impact:

  • Unauthorized access to SSL-VPN resources
  • Potential lateral movement within corporate networks
  • Compliance violations for session management requirements
