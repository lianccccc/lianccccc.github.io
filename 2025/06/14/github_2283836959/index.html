PHP FPM 远程代码执行漏洞

漏洞信息

漏洞名称： PHP FPM 远程代码执行漏洞

漏洞类型： 命令执行

漏洞等级： 高危

漏洞描述： 该漏洞存在于PHP FPM中，攻击者可以通过构造特定的请求，利用Metasploit框架中的exploit/multi/http/php_fpm_rce模块实现远程代码执行。由于新的PHP编码器”HEX”导致payload大小超出限制，使得该漏洞在当前Metasploit版本中不再有效。

产品厂商： PHP

产品名称： PHP FPM

影响版本： 7.2.19

来源： https://github.com/rapid7/metasploit-framework/issues/20310

类型： rapid7/metasploit-framework:github issues

来源概述

Steps to reproduce

How’d you do it?

  1. Victim: PHP 7.2.19, on Nginx 1.14.2 with FPM
  2. Attacker: Metasploit Framework v6.4.45-dev, using module exploit/multi/http/php_fpm_rce

-> Exploit worked reliable on older Metasploit versions

Expected behavior

Default settings, default target (PHP), default payload (meterpreter/reverse_tcp)
During check phase, the max payload length is determined being 1760 bytes
Payload gets sent and a meterpreter shell is opened

Current behavior

During check phase, max payload length is determined being 1760 bytes
Then payload is prepared and encoded via the new default PHP encoder “HEX”, which is larger (2270 bytes), making the exploit fail

Metasploit version

v6.4.45-dev
